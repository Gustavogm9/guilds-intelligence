🤖 Onde a IA é Executada — Todos os Pontos

⚠️ Importante: Os scripts Python (gerar_relatorio_cliente.py, gerar_social_media.py) não consomem tokens — são apenas formatação/renderização. Todos os tokens são consumidos pelo Claude Agent durante as fases de pesquisa e síntese, antes de chamar os scripts.

🔍

Web Search — buscas em tempo real

Cada busca consome tokens de input (query) e de output (resultados processados). O agente lê e interpreta os resultados, o que gera consumo significativo.

~1.000–1.500 tokens por busca
🧠

Síntese e Análise dos Resultados

Claude lê todos os resultados das buscas, identifica padrões, elimina ruído, e estrutura os insights em categorias. É a etapa de maior consumo de contexto.

~8.000–15.000 tokens de input + ~5.000–8.000 output
📝

Escrita do Conteúdo dos Relatórios

Claude gera o texto de cada seção do relatório: fatos, análises, recomendações, impacto para o cliente, mensagem da Guilds. Texto estruturado de alta qualidade.

~3.000–6.000 tokens de output
🗺️

Mapeamento de Nichos por Cliente

Dado o perfil do cliente, Claude identifica quais nichos impactam o negócio direta e indiretamente, com justificativas. Acontece uma vez por cliente (ou quando o perfil muda).

~2.000 tokens input + ~500 tokens output
👤

Extração de Perfil de Cliente (texto livre → JSON)

Quando você passa dados do cliente em qualquer formato, Claude lê e estrutura o profile.json. Acontece apenas uma vez no onboarding de cada cliente.

~2.000 tokens input + ~1.000 tokens output
🗣️

gTTS — Áudio MP3 (SEM tokens)

Google Text-to-Speech API gratuita. O roteiro é gerado pelo Claude (conta nos tokens acima), mas a conversão em áudio é feita pelo gTTS — sem custo adicional.

🚧 O Que Ainda Não Foi Construído

🖼️ Logo do cliente nos cards

Cards ainda usam apenas o branding da Guilds. Adicionar logo_path no profile.json e integrar no gerar_social_media.py para imprimir o logo do cliente nos cards.

Estimativa: ~30 min de desenvolvimento

📧 Envio automático por email

O relatório é gerado mas precisa ser enviado manualmente. Integrar SMTP ou API (SendGrid, Resend) para entregar PDF + MP3 + cards diretamente no email do cliente.

Estimativa: ~1h de desenvolvimento

📲 Envio automático por WhatsApp

Integrar WhatsApp Business API (Twilio, Z-API, WPPConnect) para enviar o briefing e os cards automaticamente para o número do cliente.

Estimativa: ~2h + conta na API

🔄 Deep Dive via cliente

Cliente ainda não tem como pedir relatórios aprofundados diretamente. Precisa de uma interface (formulário web, bot WhatsApp ou link de email) para o cliente fazer pedidos.

Estimativa: depende da interface escolhida

📊 Painel de controle / dashboard

Não há visibilidade dos relatórios gerados, clientes ativos ou uso de tokens. Um dashboard simples em HTML ou planilha mostraria o histórico e facilitaria a gestão.

Estimativa: ~1h para dashboard simples

💳 Registro de uso para cobrança

Não há log automático de "relatórios gerados por cliente" para facilitar o faturamento. Um arquivo de log simples resolveria isso facilmente.

Estimativa: ~20 min de desenvolvimento
